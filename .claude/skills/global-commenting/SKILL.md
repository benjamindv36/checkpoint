# Global Commenting

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle global commenting.

## When to use this skill:

- When adding comments to any code file in any language
- When writing self-documenting code with clear naming and structure
- When documenting complex logic sections that need explanation
- When reviewing code to remove or update outdated comments
- When deciding whether a comment is necessary or if code clarity is sufficient
- When ensuring comments are evergreen and don't reference temporary changes
- Use this skill universally when considering code documentation and comments
